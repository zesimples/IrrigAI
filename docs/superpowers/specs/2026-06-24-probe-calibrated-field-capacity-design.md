# Probe-calibrated field capacity ‚ÄĒ design

**Date:** 2026-06-24
**Status:** Implemented (2026-06-24) ‚ÄĒ see docs/superpowers/plans/2026-06-24-probe-calibrated-field-capacity.md
**Scope:** Backend engine (Phases 1‚Äď2 of the recommendation-quality fix)

## Problem

Nearly every sector reports *"100% da √°gua dispon√≠vel"* and the engine almost never
recommends irrigation. Confirmed root cause (live data + live engine run):

`engine/water_balance.py:build_water_balance` clamps probe SWC to the configured
field capacity (`swc = max(pwp, min(fc, swc_probe))`) and `compute_depletion` floors
at 0. FC is sourced from generic soil-texture presets (sandy_loam = 0.16,
clay_loam = 0.30, default 0.28) via plot/SCP, but the VWC sensors read on their own
absolute scale (typically 0.30‚Äď0.45 m¬≥/m¬≥) ‚ÄĒ far above those preset values. So SWC
pins to FC, depletion = 0, and `trigger.should_irrigate` never fires.

**Evidence:**
- T63 ‚Äď Campo Ampelogr√°fico: preset FC 0.16, probe VWC avg 0.445 ‚Üí live run: SWC
  clamped to 0.16, Dr = 0.0 mm, action = skip, "100% da √°gua dispon√≠vel".
- S2 ‚Äď Non Pareil: FC 0.27 (within sensor range), probe 0.173 ‚Üí Dr = 96.9 mm,
  action = irrigate. Proves the engine works when FC matches the sensor scale.
- Blast radius: of 76 VWC probe sectors, **18 are permanently pinned** (even the
  14-day minimum VWC ‚Č• FC, so depletion is mathematically always 0) and **28** have
  average VWC ‚Č• FC.

**Why existing auto-calibration doesn't fix it:** `engine/auto_calibration.py`
already computes `observed_fc_pct` / `observed_refill_pct` from irrigation-cycle
peaks/troughs, but its final step snaps them to the *nearest existing soil preset*
("NEVER creates custom soil values") and it is a manual opt-in API nobody has
accepted. A sensor reading 0.33 in textbook "sandy_loam" can never match a 0.16
preset, so it is structurally unable to set FC to the sensor's true drained upper
limit.

## Goal

Feed the water balance soil reference points (FC and refill line) calibrated to each
probe's own VWC scale instead of generic soil-texture presets. The engine's
depletion/trigger logic is unchanged ‚ÄĒ it becomes correct once the numbers match the
sensor.

## Design

### 1. Calibration computation (`engine/auto_calibration.py`)

Reuse the existing cycle analysis (`compute_observed_reference_points`): cycle peaks ‚Üí
`observed_fc`, troughs ‚Üí `observed_refill`.

Add an **envelope-proxy fallback** for sectors with < 3 clean irrigation cycles:
- `observed_fc` = high percentile (~95th) of VWC over the trailing window.
- `observed_refill` = low percentile (~10th) of VWC over the trailing window.

Return a confidence tier:
- `cycles` (high) ‚ÄĒ ‚Č• 3 clean cycles.
- `envelope` (medium) ‚ÄĒ percentile fallback.

Plausibility guards (reject ‚Üí no calibration for that sector, fall back to preset):
- `observed_fc` must be within [0.10, 0.60] m¬≥/m¬≥.
- `observed_fc ‚ąí observed_refill ‚Č• ~0.03` m¬≥/m¬≥ (minimum usable spread).

### 2. Persistence ‚ÄĒ `probe_calibration` table

One row per sector:

| column | type | notes |
|---|---|---|
| `sector_id` | FK ‚Üí sector | unique |
| `observed_fc` | float (m¬≥/m¬≥) | drained upper limit |
| `observed_refill` | float (m¬≥/m¬≥) | refill / lower bound |
| `method` | str | `cycles` \| `envelope` |
| `num_cycles` | int | cycles used (0 for envelope) |
| `consistency` | float | 0‚Äď1 (from cycle stdev) |
| `window_days` | int | trailing window analysed |
| `computed_at` | datetime | recompute timestamp |

SQLAlchemy model + autogenerated Alembic migration (`make makemigration`).

### 3. Scheduler job ‚ÄĒ `recompute_probe_calibration`

Weekly, per-farm, Redis-locked ‚ÄĒ mirrors the existing reference-recompute job in
`services/scheduler.py` (`_run_per_farm_job` / `classify_per_farm_run`). Computes and
upserts `probe_calibration` rows.

### 4. Pipeline consumption (`pipeline.build_sector_context`)

FC / refill resolution precedence:
1. SCP explicit `field_capacity` (user override) ‚ÄĒ highest, unchanged.
2. **`probe_calibration` `observed_fc` / `observed_refill`** ‚ÄĒ NEW.
3. plot / preset FC.
4. hardcoded default 0.28 ‚ÄĒ last resort.

Both bounds are calibrated together: `FC = observed_fc`, lower bound =
`observed_refill`. This is essential ‚ÄĒ calibrating FC *alone* while leaving preset
PWP = 0.07 would balloon TAW and the trigger would still rarely fire. Using the
observed refill line as the lower bound shrinks TAW to the real operating band, so
the depletion-vs-RAW trigger tracks the grower's actual irrigation cycle.

Track which source won (for `swc_source` and `defaults_used`).

### 5. Water balance / snapshot

- `swc_source = "probe_calibrated"` when calibrated bounds are used.
- Record `observed_fc` / `observed_refill` / `method` / `consistency` in
  `inputs_snapshot` (alongside the existing `swc_model` metadata) so the calibration
  is auditable and a human can override it.
- The clamp / depletion math is unchanged.
- Per-sector `try/except` ‚Üí fall back to preset on any error (mirrors the
  `soil_water_model` integration pattern in `pipeline.run`).

### 6. Trigger

Untouched. Phase 3 (replacement-based daily-irrigation paradigm) is deferred.

## Data flow

weekly `recompute_probe_calibration` job ‚Üí `probe_calibration` rows ‚Üí daily
recommendation pipeline reads them in `build_sector_context` ‚Üí water balance uses
calibrated bounds ‚Üí trigger fires realistically ‚Üí recommendation records
`swc_source = probe_calibrated` + the observed numbers.
